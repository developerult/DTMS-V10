Missing Fee Approval Set Up

When a fee comes into TMS for the first time via EDI or if it has not been configured as part of the carrier’s fee tolerance settings, users can configure the fee for automation so the next time this fee comes in, it will follow the business rules for automation and approval. Carriers who are entering invoices into the portal, should contact logistics if you do not see the fee you’re trying to enter. Users should update the carrier configuration for fee to display for carrier. (See Carrier Page-Accessorials)

Click on Add Config.

The approval configuration screen will appear for users to configure accordingly.

Auto Approve = Check the box to have this fee auto approve based on tolerance settings. Otherwise, users will need to approve this fee next time if it was not included during dispatch/tender.
Allow Carrier Updates = Check the box to allow carrier edit/updates (via Portal or EDI) Recommended.

Show Contracted Fees = Check the box to display fees during invoice entry (via Portal). Recommended.

Then select your tolerance settings-

Select only one.

BLUE UNDERLINE -Insert dollar value low and high. For example, $0 (low) between $100 (high). You accept anything between $0 - $100 for this fee. Anything else fails audit tolerance, requires approval.

RED CIRCLE– Check Dynamic Expected Cost to insert a percentage low and high range. For example, 100% (% below) to 25% (% above). Will approve $0 to 25% higher than the cost.

 

Or

Set your expected cost at $50. Your percentage low set to 20% (which is $50-$10=$40) to 20% above ($50 + $10 = $60) above. This fee expected price range is between $40 - $60. Or just insert expected cost and no low/high range.
