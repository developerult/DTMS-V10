Step 1 - Go to Fee Approval Page

 

Each implementation can have a different configuration based on automation of the fee approval process. Some fees will be auto approved, and others will require the attention of a TMS user to review/edit/approve the fee values. This automation configuration and tolerances can be viewed in the Carrier Page under each carrier in the Accessorials tab.

Select a carrier or show all carriers to review each carriers’ fees needing approval.

The system will return all transactions for selected carrier-NGL Trans. This example is where a carrier inserted a Tarp Fee at time of invoicing and but was not included during the tendered ($0) process.

It will display the SHID (ShipID=CNS number or specific carrier assigned Pro Number)
Order Number = shippers order number
Pro Number = unique number assigned by TMS and not the same as carrier assigned pro number.
CNS Number = unique number assigned by TMS and can be the ShipID if no carrier assigned Pro Number is assigned.
Amount = Amount the carrier inserted or EDI fee received.
Tendered = is the value (cost) that was tendered (pre-assigned).
Caption = Fee name

 

 

Example of Tendered fee and Fee Amount entered by carrier.


Mod Date = Date entered by carrier or received by EDI.
Dest Name = Ship to location
Lock All Costs = If checked, this was to preserve any costing from being over written. To approve fees, this will need to be unchecked. The system will prompt a message indicating costs are locked. This is typically done on Spot rate or API carrier shipments.
Lock BFC = Same as above if using BFC functionality. BFC stands for Benchmark Freight Cost.
Message = System generated message that will indicate to user why the fee needs approval.

 

 

 

Approve = when users click on Approve, you are agreeing to the amount entered by the carrier. This will update the fee(s) amount in the Revenue Screen and re-calculate and update the carrier cost. Then push this invoice through the audit function.

To Approve Fee = Click APPROVE. 
Note, if there are multiple fees for this invoice needing approval, all fees will need to be approved before it will be sent through the audit process. Each fee is processed separately.

 

Edit = User is modifying the carrier fee entry and is approving the lessor or higher amount entered by the carrier. This will update the fee(s) amount in the Revenue Screen and re-calculate the carrier cost. Then push this invoice through the audit function.

 

To Edit Fee click EDIT.
Insert approved fee value, click save.

Click APPROVE.

 

This approval process must be done for each carrier.
