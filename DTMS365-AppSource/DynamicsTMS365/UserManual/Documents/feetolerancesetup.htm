Fee Tolerance Set up


The next step would be to apply your tolerance ranges. We have three different settings. First is a flat dollar low and high range. Second, you can apply a percentage low and high range against a known value. Third, is Dynamic Expected Cost. If no known value is entered, the percentage low and high ranges will compare against the fee entry.

1.  Each Fee for each Carrier has individual settings assigned to each Legal Entity.

a.  Flat Rate Low Value: minimum dollar value allowed; typically, zero.

b.  Flat Rate High Value: maximum dollar value allowed.

c.   Expected Cost:  The Expected Cost is only used with the percentage tolerance calculations.

d.  Lowest Percentage Factor:  based on a pre-determined Expected Cost.  The lowest percentage below the Expected Cost that is allowed, typically this is 100%, which will evaluate to a minimum cost of zero dollars.

e.  Highest Percentage Factor: base on the Expected Cost, the highest percentage above the Expected Cost that is allowed.

2.  Calculation Rules:

a.  If the Expected Cost is zero, the system will use the Flat Rate values to determine if the Cost falls inside the Flat Rate Tolerances.

b.  If the Expected Cost is non-zero, the system will use the Percentage Factors to determine if the Cost falls inside the Percentage Tolerances.

 

3.  Flat Rate Formula:

a.  If Cost falls below the Flat Rate Low Value the fee cannot be auto-approved.

b.  If the Cost exceeds the Flat Rate High Value the fee cannot be auto-approved.

4.  Low Percentage Factor Formula: If the Cost falls below the Calculated Low Value based on the Low Percentage Factor the fee cannot be auto approved.

a.  If Low Percentage Factor is zero, the Calculated Low Value is equal to the Expected Cost; the fee cannot be auto approved if it falls below the Expected Cost.

b.  If Low Percentage Factor is greater than zero, we apply the Low Percentage Factor to the Expected Cost:  where the Calculated Low Value equals the difference between the Expected Cost and the product of the Expected Cost and the Low Percentage Factor.

c.  Example:  In this example the Lowest Cost Allowed is $105.00

i. Expected Cost = 150, Low Percentage Factor = 30%. 

ii.          Calculated Low Value = 150 - (150 * 0.30)

iii.         or $105.00 = $150 - $45.00

5.  High Percentage Factor Formula:  If the Cost exceeds the Calculated High Value based on the High Percentage Factor the fee cannot be auto approved.

a.  If the High Percentage Factor equals zero the Calculated High Value is equal to the Expected Cost, the fee cannot be auto approved if it exceeds the Expected Cost.

b.  If the High Percentage Factor is greater than zero, we apply the High Percentage Factor to the Expected Cost: where the Calculated High Value equals the sum of the Expected Cost and the product of the Expected Cost and the High Percentage Factor.

c.  Example:  In this example the Highest Cost Allowed is $195

i. Expected Cost = 150, High Percentage Factor = 30%. 

ii.          Calculated High Value = 150 + (150 * 0.30)

iii.         or $195.00 = $150 + $45.00

Using Flat Rate example:
Enter in dollar low and high range. Low = $0       High = $195

The fee entered by the carrier must fall between this range.
Percentage values and average value all must be zero and are not being applied.



 

 

Using Percentage Rate Example:
Dollar value ranges are equal to zero and not being applied.

Percentage values are set to 100% below Expected Cost value (of $150) and 30% above (.30 x $150 - $195) Expected Cost Value $150.

Note: 100% below $150 means $0 cost.


Dynamic Expected Cost
This is used when you want to allow for variability in the fee entry by the carrier. When checked on, you enter in the % Below value (100% to equal zero) and % Above value ranges.

For example: Let’s say you configured fuel to use Dynamic Expected Cost. If your settings were 100% below value and 10% high value and the actual fuel cost for a given load was $225.00. This configuration will allow for $0 on the below value and will allow up to $247.50 (10%) on the high value.
