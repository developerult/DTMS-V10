Validation

If Validation is turned off at the Resource level that means the Carrier Automation algorithm
will not consider the resource at all.
If Validation is on at the Resource level, then the system will look at the rules set up via the Overrides Tab.

 

Overrides Tab
1) If the rule is off it will not be validated by the validation algorithm (Warehouse only)
2) If either or both of Require Reason Code or Password is on then if validation fails the user can
perform the action in order to override and continue to book the appointment (Warehouse only)
3) If validation is on for the rule BUT neither Reason Code or Password is checked then if the validation fails the user cannot do anything to override
