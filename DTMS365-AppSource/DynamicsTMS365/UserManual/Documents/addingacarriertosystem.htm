Adding a Carrier to System


Here users can add new carriers, manage carrier contacts, add accessorials by carrier.

If your configuration uses Dynamics NAV for the ERP integration, go to NAV and in the Transportation, Department click Set Up. If using different ERP integration, skip this step and manually enter in your carrier in the desktop application as normal.

Select Dynamics TMS Vendor Carrier

Click on New in the action’s menu and search for the carrier you want to send to TMS. Be sure to tab through and click ok when done. Click Refresh and you should see the check box marked. Indicating this is now synchronized and sent to TMS for further set up.

Then login into DTMS and go to the Carrier Page for the specific Legal Entity you want to assign this carrier to. Use Change Legal Entity to load new carrier for different Legal Entity.

Click on Add New Carrier and using the drop-down list find the new carrier to add.

 

Select the Dispatch Type for each carrier.

If using TMS tariff system, select NGL Tariff.
If using API’s, contact NGL for set up and Account Reference information and further API configuration. Users can always reference a carrier that is already using API’s to obtain this information.

 

Carrier Account Reference =Is used for API’s only. See support for the correct reference account.

Show Audit Fail Reason = If carrier is set up to use Web Settlement, it will display Failed if invoice failed audit.

Show Pending Fee Fail = If carrier is set up to use Web Settlement, it will display Pending waiting on approval.

Expired loads is used for web tendering and the time settings used as to when a load tender should expire. There are two settings. 1) Legal Entity (default) 2) By carrier.

If new carrier supports multi location pick up LTL consolidation, check Allow LTL Multi-Pick Consolidations. The system will calculate freight costs based on the aggregated weight from each location, shipping to the same delivery location. Additional set up in the Parameters is required. See support for assistance.

 

Expired Loads = Check this box to use the Legal Entity default (See Legal Entity setup). Or leave blank for carrier specific.

Insert the TMS distribution email address created for the logistics team. Any expired loads will be sent to this distribution list.

Expired Loads Email To = Email address used to receive expired load tenders. Best to use group TMS email address.

Expired Loads Cc = Carbon copy Email address used to receive expired load tenders.

 

Then insert the amount of time in minutes you allow for carriers to accept/reject a load tender. If Legal Entity default is checked, it will use setting in the Legal Entity Maintenance screen.

Carrier Accept Load Mins = The amount of time in minutes carriers have to accept a load tender. This can be set at the Legal Entity level or carrier specific level. If Legal Entity is checked, the system will use the Legal Entity default setting. Note, this does not impact API carriers.

Once you have added the carrier, click in the row of the carrier you just entered so it’s highlighted.

Set Billing Company allows to you assign the company/location where all freight bills or related information should be sent i.e. Bill To. Click and highlight a carrier so it turns Blue, Select Set Billing Company in the Actions menu and select the company/location.

In the Actions Menu, click on Set Billing Company and select the company to display the billing company address for freight bills.
