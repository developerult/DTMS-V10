Legacy Accessorials

Stop charges, pick charges, fuel surcharges have specific allocation rules that should not be modified.

Legacy Stop fee:

•      Any accessorial (codes 20-28) like stop X fee is considered a legacy stop fee

•      legacy stop fees use specific variable codes formulas 7-16 with names like Stop X Flat fee. These variable codes settings should match the accessorial codes settings with the same stop # For Example, stop 1 fee should be assigned the accessorial code formula of stop 1 flat fee (current v6.0 system logic)

•      legacy stop fees are always allocated by destination, so the fee allocation type is ignored. The system will reset this value when costs are recalculated if it is changed.

•      legacy stop fees are always allocation type of destination and always have allocation formula of weight. (user settings are ignored or will be modified by the system to reflect default values) The system will reset this value when costs are recalculated if it is changed.

•      fee calculation type for stop fees is always set to unique

•      stop fees are adjusted for rounding errors automatically with any discrepancy being applied to the order with the heaviest weight at that specific destination

•      The allocated cost will be stored in the fee value

•      The system will use the highest configured stop fee available if there are additional stops that do not have fees configured in the carrier tariff contract.  For example, if you have 4 stops configured but build a 5 stop load, the system will use the stop 4 fee$ amount for stop 5.

Legacy Fuel Surcharge Fee (The standard Rate Per Mile and Rate Percent fuel fees)

•      The Fee Allocation Type is always Load.  The system will reset this value when costs are recalculated if it is changed.

•      The Fee Calculation Type is always Unique.  The system will reset this value when the costs are recalculated if it is changed

Legacy Pickup Fee (The standard Pick 1, Pick 2 etc… fee):

•      The Fee Allocation Type is always Origin.  The system will reset this value when costs are recalculated if it is changed.

The Fee Calculation Type is always Unique.  The system will reset this value when the costs are recalculated if it is changed
