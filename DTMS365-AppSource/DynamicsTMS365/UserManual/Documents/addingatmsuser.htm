Adding a TMS User


Only Admins can add users.

Go to the System page and click on User Maintenance.

Here you will see a list of current users for this legal entity. To change Legal Entities, click Change Legal Entity and make Legal Entity selection.

Click on Add New User

The Add New User dialog box will open.

 

1) Enter in the user’s email address and information.

2) Enter in desired username (best to follow pre-assigned naming convention like domain name of (NGL\JLee).

3) Check the box to Auto Generate Password. The system will send temporary password to user’s email address. Users can change once they’ve logged in under settings.

If you do not want the system to auto generate password, leave the box unchecked and you can manually insert a password (recommended). Then check the box Send Password to User.

4) Select User Group (Role based). Each user group will have selected features based on the user group they’re assigned to. Note: Internal users should never be assigned as a Super User. This is reserved for TMS Support. Carrier and Carrier Accountants are for Carriers only.

5) Check NGL API if wanting to provide access to API rate shopping. Insert API account code (see support for more information and if your implementation supports API’s.)

6) Click on ADD to assign this user access to a specific carrier(s) freight bill settlement screen, as if they’re entering in freight bills on behalf of carrier or for review purposes. Not intended for all users.

7) Click Save.

It’s best to test the users credentials prior to handing out their username and password for the portal to ensure of the proper settings and access.

CSR/Sales = Have access to rate shopping and NEXTrack ALL page to track shipment status.
Warehouse = Have access to managing the scheduling system, NEXTrack All page.

Administrators = have access to the entire functional TMS suite and system configuration settings in the portal.

Operations = have access to functional TMS suite.

Once you have created the user, that user will automatically be associated with the legal entity where you added the new user.

Now add the Single Sign On accounts for the Carrier API’s and Bid Portal. This process only pertains to TMS users who will rate shop and/or perform load postings. Otherwise, this user is complete.
