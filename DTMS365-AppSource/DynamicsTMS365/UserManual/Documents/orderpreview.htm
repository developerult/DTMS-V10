Order Preview


The Order Preview page is an exception screen that will capture all order exceptions for each warehouse location. In most implementations, this will include an order modification i.e. customer added more product to the order or date changes.

Any order that already exists in TMS and is not booked (finalized) with a carrier the system will automatically bring in the changes into TMS.

If the order has already been accepted and booked with a carrier (PB-Finalized) status, the order modification would come to this screen with a Warning Message FINALIZED. This is a warning to the user that this order already exists in the TMS and is booked with a carrier.

Another scenario would be if an order was shipping from one origin location and now customer service modifies an order to ship from a different location. The warning message would read NEW COMP.

NO PRO means a new order; however, most implementations are configured to bring in new orders automatically.

Users should unfinalize (by going to the Booking Options tab in Load Board) the order (PB to P) that already exists in TMS and then bring in the order modification by clicking on the black arrow. The system will recalculate the costs for the carrier assigned and users should re-dispatch the load to the carrier as necessary, so the carrier has the updated information. If using an API carrier, users should re-rate the shipment and dispatch again. 

Users can apply filters by Company (location) or Trans Type (FOB Delivered or CPU’s)

Users can click on View Import Changes in the Actions section to identify the specific changes to the highlighted order.
