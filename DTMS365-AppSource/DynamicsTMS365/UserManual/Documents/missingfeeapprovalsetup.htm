Missing Fee Approval Set Up

When a fee comes into TMS for the first time and it is not configured as part of the carrier’s fee tolerance settings, users can configure the fee for automation so the next time this fee comes in, it will follow the business rules for automation and approval.

Auto Approve = Check the box to have this fee auto approve based on tolerance settings. Otherwise, users will need to approve this fee next time.
Allow Carrier Updates = Check the box to allow carrier updates (via Portal or EDI) Recommended.

Show Contracted Fees = Check the box to display fees during invoice entry (via Portal). Recommended.

 

Then select your tolerance settings-Select only one.

BLUE -Insert dollar value low and high. For example, $25 (low) between $100 (high). You accept anything between $25 - $100 for this fee.

RED – Check Dynamic Expected Cost to insert a percentage low and high range. For example, 100% (% below) to 25% (% above)

Or

Set your Expected Cost at $50. Your percentage low set to 20% (which is $50-$10=$40) to 20% above ($50 + $10 = $60) above. This fee expected price range is between $40 - $60.

 

Go to the Fee Approval Screen and approve any waiting fees entered by a user or carrier. You will need rights to access this page. Click on Edit to change the amount you want to approve or click Approve.

You can filter the data to see fees by carrier etc. Additional filters can be applied. You will see every fee entered by a TMS user, carrier, or EDI for a given load that needs approval in this screen. It might be helpful to group the data by ShipID (CNS Number) to see fees associated to the same ShipID (invoice).

For example:

Filter by: CNS106722
Any of the column headers such as CNS Number by clicking and dragging the header just above its location. If you click on the group label, it will sort ascending / descending if you hit it a second time. The third time will remove the sort.

Users should review each fee by carrier and either Approve or Edit the fee.

Approve = when users click on Approve, you are agreeing to the amount entered by the carrier. This will update the fee(s) amount in the Revenue Screen and re-calculate and update the carrier cost. Then push this invoice through the audit function.

Edit = User is modifying the carrier fee entry and is approving the lessor or higher amount entered by the carrier. This will update the fee(s) amount in the Revenue Screen and re-calculate the carrier cost. Then push this invoice through the audit function.

Click Save when done.

The system will display what the carrier entered as a fee value versus what the tendered cost for this predetermined fee.

Locked Costs Approval

If the cost box is locked, the system will prompt the user a warning messaging indicating that this is a fee associated with a Locked Cost load (Spot or API carrier) and have been locked to protect from automated updates. All Spot Rate/API loads are “locked” in TMS to prevent automated updates. Users can also see which loads have locked costs by the display box. If checked, Costs are locked.

Before any fees can be approved, the system will notify the user that costs are locked and must click OK in order for the system to unlock the carrier costs automatically, allowing the system to insert the fee update to the Revenue Screen in TMS and recalculate costs.

After invoice has been entered. The system will update the status of the invoice based on the system configuration.

 

There are 3 status workflows:

1) AA = Approved. The system automatically approved the entry, passed audit, and will disappear from the screen. Setting the PExp (Pay Expense workflow to AA-auto approved) in the Load Detail Screen.

There are three ways to view AA statuses:

A) Go to Load Board (Web Client) and apply the filter called Pay Code = AA for a record.

B) Go to the Load Detail Screen (Desktop Client) for this load. The PExp will be in AA status.

 

C) Go to AP Freight Bill Audit Approval Screen (Web Client) and apply the filter called Pay Code = AA

          a. Apply your filter by carrier.

          b. Select a date range based on invoice date.

          c. Use drop down and select Approved to see all approve freight invoices for this date range.

 

 

2) PA = Pending Approval. The system accepted carrier web/EDI entry and is waiting on TMS user approval. This transaction will remain in the Pending Approval Page until a user approves them.

There are three ways to view PA statuses:

A) Go to Load Board (Web Client) and apply the filter called Pay Code = PA

The status will be updated in TMS on all loads that are Pending Approval (PA). Setting the PExp (Pay Expense workflow to PA-Pending Approval) in the Load Detail Screen.

B) Go to the Load Detail Screen (Desktop Client) for this load. The PExp will be in PA status.

C) Go to AP Freight Bill Audit Approval Screen (Web Client)

          a. Apply your filter by carrier.

          b. Select a date range based on invoice date.

          c. Use drop down and select PA (Pending Approval) to see all Pending Approval freight invoices
              for this date range.

3) M = Match/Failed.  The system accepted the freight invoice entry as a matched transaction in TMS but has failed the audit process. All transactions will need to be fixed, re-audited, and pass audit to be set to AA status.

There are three ways to view M status transactions:

A) Go to Load Board (Web Client) and apply the filter called Pay Code = M

B) Go to the Load Detail Screen (Desktop Client) for this load. The PExp will be in M status.

C) Go to AP Freight Bill Audit Approval Screen (Web Client) and apply the filter called Pay Code = M

          a. Apply your filter by carrier.

          b. Select a date range based on invoice date.

          c. Use drop down and select M (Matched) to see all failed invoices for this date range.

NOTE: All transactions will need to be in AA status, indicating the invoice passed audit and has been sent to the ERP system for payment.
